# LMS Lesson Experience Redesign Requirements

I want to redesign the lesson experience for my Learning Management System (LMS) using modern UX, UI, and instructional design best practices.

The LMS should support any subject (programming, business, marketing, finance, mathematics, science, etc.), so the lesson structure must be generic and reusable.

## Core Requirements
- Remove the video-first approach completely.
- Lessons should be text-based, engaging, and easy to scan.
- Follow modern e-learning principles used by platforms like Microsoft Learn, Coursera, Khan Academy, Codecademy, freeCodeCamp, Google Developers, and LinkedIn Learning.
- Focus on active learning instead of passive content consumption.
- Each lesson should take approximately 5–10 minutes to complete.
- Break content into small, digestible sections (microlearning).
- Minimize long paragraphs. Use headings, cards, callouts, bullet lists, examples, diagrams, and interactive elements.
- Design for desktop, tablet, and mobile.

## Lesson Structure
Every lesson should follow this structure:
1. Lesson title
2. Estimated completion time
3. Difficulty level
4. Learning objectives
5. Short introduction
6. Key concepts (broken into sections)
7. Visual explanations (icons, diagrams, illustrations, timelines, comparison tables where appropriate)
8. Real-world examples
9. Interactive examples
10. Tips and best practices
11. Common mistakes
12. Knowledge check (multiple choice, true/false, matching, etc.)
13. Reflection or mini activity
14. Summary
15. Key takeaways
16. Next lesson navigation

## Content Presentation
Avoid long walls of text. Instead use:
- Information cards
- Expandable accordions
- Highlight boxes
- Warning boxes
- Tip boxes
- Examples
- Tables
- Comparison layouts
- Process diagrams
- Step-by-step sections
- Icons
- Checklists
- Progress indicators

## Interactive Learning
Include opportunities for learner interaction throughout the lesson, such as:
- Quick quizzes
- Reflection questions
- Drag-and-drop activities (where applicable)
- Fill-in-the-blank exercises
- Scenario-based questions
- Practice exercises
- Mini challenges
- Self-assessment checkpoints

**Do not wait until the end of the lesson to engage the learner.**

## Navigation
The lesson page should include:
- Course progress
- Lesson progress
- Sidebar showing all lessons
- Completed lesson indicators
- Current lesson indicator
- Previous and Next buttons
- Breadcrumb navigation
- Optional table of contents for longer lessons

## Visual Design
Create a clean, modern interface with plenty of whitespace. The design should prioritize readability by using:
- Clear typography hierarchy
- Consistent spacing
- Rounded cards
- Soft shadows
- Accessible color contrast
- Responsive layouts
- Sticky navigation where appropriate

**Avoid clutter.**

## Accessibility
Follow WCAG accessibility best practices. Include:
- Keyboard navigation
- Screen reader support
- Semantic HTML
- High contrast
- Descriptive labels
- Focus states
- Responsive font sizing

## Learning Experience
- The learner should never feel overwhelmed.
- The page should feel like a guided learning experience rather than reading a textbook.
- Content should alternate between: **Reading**, **Looking**, **Thinking**, **Interacting** to maintain engagement.

## Technical Requirements
- Use semantic HTML5.
- Use reusable components.
- The design should be component-based so it can work with any future course.
- Avoid hardcoding content specific to HTML, CSS, or programming.
- The lesson template should work equally well for: Programming, Business, Accounting, Marketing, Sales, Finance, Project Management, Data Analytics, Mathematics, Science, Soft Skills, Compliance Training.

## Goal
Create an LMS lesson experience that feels modern, professional, highly engaging, and optimized for learning retention rather than simply displaying information. It should be scalable for thousands of lessons across different subjects while maintaining a consistent, intuitive user experience.
